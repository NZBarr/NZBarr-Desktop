Manuals

Practical guides for setting up NZBarr and using it day to day.

These notes are written to help testers and early users understand the flow without having to read the code or guess how the app wants to be used.

01

Getting Started

Install NZBarr, open the app, and add your first NZB files or browse the existing library.

  1. Set your downloader and license settings first.
  2. Add a few releases and confirm they appear in Browse.
  3. Open a release detail page to inspect metadata and artwork.
  4. Use the edit modal when a release needs correction or relinking.

02

Smart Preparation

Before importing, NZBarr can clean the release name while keeping useful descriptors that you intentionally placed in brackets.

  • Technical tags such as codec, resolution, source, and season markers are normalized.
  • Human-readable descriptors stay in the filename if you keep them in brackets.
  • Title matching is tolerant of common variants like ampersands, apostrophes, and possessives.

03

Browsing and Editing

The Browse page is where you inspect releases, open linked media, and correct metadata when something was matched incorrectly.

  • Click a release row to open the release detail page.
  • Use the linked-media button to jump to the movie or TV record.
  • Open the edit modal to adjust IDs, titles, or raw MediaInfo when needed.

04

Drag & Drop Import

You can also import by dragging NZB files onto the app. That is the quickest way to add files when you already have them in a folder or on the desktop.

  • Drag one or more NZB files into the app window.
  • NZBarr will import them the same way as a file picker import.
  • This works best when the filenames already contain useful tags and a clear title.

05

Auto Refresh

Auto Refresh re-uploads older NZBs so they can stay available and can be re-processed when the release is aged.

  • Set the minimum age threshold in years.
  • Choose how often NZBarr checks for eligible releases.
  • Configure the refresh folder, ngPost path, and archive password in settings.
  • Use the notification option if you want a visible completion message.

06

Re-analyze and Metadata Recovery

If a release has incomplete metadata, use re-analyze to download only a capped sample and then recover as much as possible from the file or filename.

  • The download cap prevents re-analysis from pulling the full release.
  • If the sample is not enough for MediaInfo, NZBarr falls back to filename tags.
  • Archive passwords from the release are used when unpacking downloaded parts.

07

Troubleshooting

When something looks wrong, the first step is usually to check the release detail page, the raw MediaInfo field, and the refresh logs.

  • If a release seems stale, re-open it after save or refresh.
  • If MediaInfo looks wrong, clear the raw field and re-analyze.
  • If Auto Refresh cannot finish, make sure the folder paths and downloader settings are still valid.

Filename pattern

How to make drag & drop and import work smoothly.

Keep the title readable

Put the actual title near the start of the filename so NZBarr can recognize it more easily when it prepares or matches the release.

Use brackets for extras

Descriptive text that you want to keep, such as special editions or other human notes, is safest when you place it in brackets.

Let technical tags do the work

Resolution, codec, season, source, and subtitle hints are best left in the filename so Smart Preparation can recognize them and keep them consistent.

Use sensible season patterns

NZBarr understands common season and episode formats, so names like Season 2, S02, or S01E08 can all help the importer understand the file.

In short: drag-and-drop works best when the filename still contains the title plus the useful tags you want NZBarr to read. The cleaner the filename pattern, the smoother the import.

API keys

Set up TMDB and Fanart.tv before you expect artwork.

TMDB API key

NZBarr sends requests to TMDB with your API key for title matching, metadata lookups, and richer movie or TV details. Without it, some matching and import helpers have less data to work with.

Fanart.tv API key

NZBarr sends artwork requests to Fanart.tv with your API key to fetch extra cover, backdrop, and logo artwork. If the key is missing, the Fanart buttons stay available but will show a clear message when clicked.

Enter both keys in Settings if you want the artwork and metadata tools to feel complete. If you skip them, the app still works, but you will see fewer search and artwork options.

System requirements

The practical setup checklist.

Desktop platform

NZBarr is built for modern macOS, Windows, and Linux desktops. It is intended to run as a normal desktop application rather than a browser tab or server process.

Storage and network

Make sure you have enough free storage for the app cache, temporary analysis folders, and your media library. NZBarr also needs internet access for metadata, licensing, and downloader handoff.

Downloader access

For sending releases or running Auto Refresh, NZBarr needs a supported downloader configured with the correct host, credentials, and completed-folder path.

Helper tools

Some advanced workflows call out to helper apps or binaries such as ngPost, MediaInfo, and an archive extractor like 7-Zip or WinRAR / UnRAR.

External apps

What NZBarr works best with.

SABnzbd or NZBGet

Point NZBarr at SABnzbd or NZBGet so it can hand releases off cleanly, track completed downloads, and refresh older NZBs end to end.

ngPost

Needed for the Auto Refresh re-upload workflow. NZBarr sends prepared post data to ngPost so it can create and post refreshed NZBs back to Usenet.

Download ngPost

MediaInfo

NZBarr calls MediaInfo for detailed stream analysis during re-analyze and refresh workflows so it can fill in codec, audio, and container metadata.

Download MediaInfo

7-Zip or UnRAR

Helpful for archive-style releases. NZBarr can unpack downloaded samples more reliably when 7-Zip or WinRAR / UnRAR is available on the system.

The core app is still easy to explore on its own, but these are the external tools that unlock the deeper workflow: sending, refreshing, unpacking, and richer media analysis.

What to configure first

The short setup checklist.

Downloader

Enter your preferred downloader settings and point them at SABnzbd or NZBGet so NZBarr can hand releases off cleanly when a user wants to send or refresh content.

Auto Refresh

Set the completed folder, refresh mode, and archive password, then point NZBarr at ngPost so the refresh workflow can run end to end.

Library organization

Make sure your local media records are linked properly so the browse pages and detail pages can show the right movie or TV information.

Manual testing

Use a few known releases to test import, edit, re-analyze, and refresh before you trust the whole library setup.
