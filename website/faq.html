FAQ

Clear answers for new users and testers.

These are the practical questions people usually ask before they install NZBarr or before they try to set it up on a new machine.

What is NZBarr?

NZBarr is a desktop app for managing NZB-based media libraries. It focuses on clean browsing, smart preparation, metadata handling, and a polished desktop workflow.

What platforms are supported?

NZBarr is built for macOS, Windows, and Linux, with attention to both Apple Silicon and Intel Macs.

Is licensing handled inside the app?

The app talks to a separate license backend. The desktop app keeps the local experience simple while the license server handles validation.

Can I use NZBarr for testing first?

Yes. The free edition is intended for testing, feedback, and getting familiar with the interface before buying a paid plan.

What if a release is linked to the wrong movie or TV show?

You can open the release detail page, use the linked-media picker, and relink it to the correct item from your local library.

Why does re-analyze not always find every field?

Some releases need only a small sample before MediaInfo can read the streams. Others only expose general file data, so NZBarr also falls back to the filename tags.

How does Auto Refresh work?

Auto Refresh looks for older releases, re-downloads them with a controlled workflow, runs MediaInfo, updates the metadata, and uploads refreshed content again.

Do I need API keys?

Yes. TMDB helps with matching and metadata, and Fanart.tv unlocks the extra artwork buttons. If the Fanart key is missing, clicking the button will show a message instead of loading artwork.

Do I need extra tools?

For the most complete experience, NZBarr works best when it can send jobs to SABnzbd or NZBGet, call out to ngPost, read MediaInfo, and unpack archives with 7-Zip or WinRAR / UnRAR. The app can be explored on its own, but these tools unlock the full workflow.

If you are testing NZBarr with a small group, the manuals page is the best next stop. It gives a clearer step-by-step overview of what to configure first and what each part of the app is for.
