How it works

A straightforward desktop workflow from import to refresh.

NZBarr keeps the app focused on the parts that matter: bring releases in, keep them organized, and recover metadata or refresh older content when needed.

Step 1

Import or browse

Start by importing NZB files or opening the existing library. Releases appear in a browsable desktop layout with direct access to their details.

Step 2

Prepare filenames

Smart Preparation can clean release names before import while preserving any descriptive text you intentionally keep in brackets.

Step 3

Link media correctly

Open the release details, inspect the linked movie or TV record, and relink the release if it was matched to the wrong title.

Step 4

Analyze or refresh

Use re-analyze when metadata is incomplete, or use Auto Refresh to keep older releases alive and re-process them end to end.

Drag & Drop

Import can happen by dropping NZB files into the app.

Fastest path

Drag-and-drop is the quickest way to import when you already have the NZB files on your desktop or in a folder.

Filename pattern matters

The smoother the filename pattern, the better NZBarr can recognize the title, season markers, and technical tags during preparation.

Keep useful descriptors

If you want to keep special notes, put them in brackets so Smart Preparation can preserve them instead of stripping them away.

Let tags stay visible

Codec, source, resolution, and season tags are most useful when they stay in the filename as readable hints for the importer.

The moving parts

What NZBarr is doing behind the scenes.

Browse and edit

The browse page is the main control surface. It shows releases, lets you open details, and gives you a place to correct metadata when a match is wrong or incomplete.

  • Click a release row to open details.
  • Use the linked-media action to jump to the movie or TV record.
  • Edit the release when you need to fix IDs or metadata.

Metadata handling

NZBarr keeps a raw MediaInfo field for deeper inspection, but it also knows how to fall back to filename tags when the sample file does not reveal enough stream data.

  • Good samples produce direct MediaInfo results.
  • Smaller or awkward samples may only expose general file data.
  • Filename tags help fill in the missing pieces when needed.

Auto Refresh

How older releases are refreshed.

Find eligible releases

The scheduler checks for releases older than your configured age threshold and queues them one by one so the refresh flow stays controlled.

Download, inspect, re-upload

NZBarr sends the NZB to SABnzbd or NZBGet, calls MediaInfo on the finished files, updates metadata, and then uploads the refreshed content back to Usenet.

Stay within limits

Auto Refresh follows your configured paths and only works with the tools you point it at, so the workflow stays predictable and portable.

Notify when done

If notifications are enabled, NZBarr shows a completion notice when a refresh finishes so you can see activity without babysitting the app.
